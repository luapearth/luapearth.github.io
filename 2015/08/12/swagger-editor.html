Swagger Editor

Aug 12 2015

You built a REST API and you need to test your endpoint, you probably open your chrome browser and add extension like POSTMAN and start testing your endpoint and suddenly you think that you also need API documentation, you end up thinking how do I document my API or what tool should I use to make the documentation.

Here's the answer Swagger Editor.

  • it's an interactive web based editor for creating API documentation and also gives you the capability to test your endpoint with the same code your writing.

To get you started here's some of the basics. 


swagger: '2.0'
info:
  version: 1.0.0
  title: Sample API
basePath: /
paths:
  /:
    get:
      responses:
        200:
          description: Ok

The code is base on YAML, correct indentation is very important.

  1. Swagger version
  2. Document Info
    • Version
    • Document title
  3. Base path of the api
  4. API paths
  5. / api url
  6. get request at the root url
  7. 200 response code
  8. Server response Ok
